# Skill: GameMaker GML

## Purpose

This skill allows the agent to create GameMaker and GML resources that connect game design, assets, objects, events and code.

## When To Use

Use this skill when the user asks for:

- a GameMaker tutorial
- GML code for a game mechanic
- beginner game prototypes
- object, room, sprite or event setup
- GameMaker debugging help
- assessment scaffolds for game projects

## Inputs

- GameMaker version if known
- GML Code or GML Visual preference
- game genre or mechanic
- required assets, objects, rooms and controls
- student programming level
- export or classroom testing requirements
- school installation or licence constraints

## Process

1. Define the playable mechanic before adding polish.
2. List required assets, objects, rooms and events.
3. Specify exactly where each code snippet goes.
4. Use small GML examples with clear names.
5. Test the mechanic in the room before adding extension features.
6. Debug through object ownership, event order, collisions, variables and the Output window.
7. Keep advanced systems optional.
8. Avoid export or SDK assumptions unless supplied.

## Output

```markdown
# GameMaker Activity

## Goal
## Assets And Objects
## Room Setup
## Event Code
## Test
## Debugging Checks
## Extension
```

## Quality Criteria

- GameMaker terminology is accurate.
- Each snippet names the object and event.
- The prototype has a clear playable goal.
- Code is readable and GML-specific.
- Installation, licence and export assumptions are avoided.
