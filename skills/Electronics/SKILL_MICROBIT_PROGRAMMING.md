# Skill: micro:bit Programming

## Purpose

This skill allows the agent to create micro:bit programming activities that combine code, device features, transfer steps and classroom safety.

## When To Use

Use this skill when the user asks for:

- micro:bit MakeCode or Python activities
- button, LED, sound, radio, accelerometer, compass or sensor tasks
- micro:bit classroom workflows
- micro:bit troubleshooting
- physical computing projects
- student worksheets or teacher notes

## Inputs

- editor: MakeCode blocks, JavaScript, Python editor or micro:bit classroom
- micro:bit version if known
- required device features or external components
- available boards, USB cables, batteries and sensors
- student programming level
- whether simulator-only work is acceptable
- safety and supervision requirements

## Process

1. Define the device behaviour students should create.
2. Identify the micro:bit feature or input/output being used.
3. Start in the simulator where practical.
4. Write or describe the code in the chosen editor style.
5. Transfer to the physical board only when needed.
6. Test behaviour on simulator and hardware separately.
7. Troubleshoot code, transfer and hardware issues separately.
8. Include saving/export steps for micro:bit classroom if used.
9. Add electrical and battery safety notes when external circuits or battery packs are involved.

## Output

```markdown
# micro:bit Activity

## Goal
## Editor
## Hardware
## Code Or Blocks
## Test
## Troubleshooting
## Safety Notes
```

## Quality Criteria

- The editor and hardware needs are clear.
- The simulator is used before hardware where useful.
- Device features are named accurately.
- Transfer and saving assumptions are realistic.
- Safety notes match the hardware.
- Code and physical faults are separated.
