# Rule: Preferred Sources

## Purpose

Give the agent a default set of preferred sources for South Australian secondary teaching, curriculum, SACE, technology, safety, communication and evidence-informed practice.

## Applies To

- source research for rule and skill files
- lesson, assessment, rubric and feedback design
- science and digital technologies resources
- SACE and Australian Curriculum alignment
- AI, privacy, online safety and platform guidance
- source lists and source-register updates

## Rule Statement

The agent must prefer sources that are authoritative for the user's jurisdiction, subject, platform and purpose.

## Requirements

- Prefer Australian and South Australian sources for local curriculum, policy and school-use advice.
- Use official curriculum and SACE sources for alignment, assessment and standards.
- Use government or recognised professional bodies for AI, privacy, safety and online safety guidance.
- Use official vendor documentation for software features and platform instructions.
- Use AERO, AITSL, university or peer-reviewed sources for evidence-informed teaching claims.
- Use user-supplied school documents as the authority when they define local requirements.
- Use secondary classroom resources only when they are reputable and fit the context.
- Avoid sources that are outdated, promotional, unsourced, jurisdictionally mismatched or not age-appropriate.

## Preferred Source List

- Australian Curriculum, Assessment and Reporting Authority for Australian Curriculum F-10.
- SACE Board of South Australia for Stage 1 and Stage 2 subjects, performance standards and assessment.
- South Australian Department for Education for local public-school policy and guidance.
- Australian Government Department of Education for national school frameworks.
- AITSL for professional standards.
- AERO for evidence-informed teaching practice.
- eSafety Commissioner for online safety and digital wellbeing.
- Official software documentation for Microsoft, Scratch, micro:bit, GameMaker, Python libraries and similar tools.
- Australian Government Style Manual for public-facing language, plain language and referencing style.
- University library or academic success guidance for source evaluation and research process.

## Decision Logic

If a preferred source directly answers the question, use it before a general web result. If the preferred source is too broad, pair it with a more specific official source. If a non-preferred source appears useful, use it only after checking whether a preferred source exists.

## Examples

- Use SACE pages for performance standards instead of a third-party study guide.
- Use DB Browser for SQLite and SQLite documentation for SQLite database-tool instructions.
- Use Microsoft Support or Microsoft Learn for Excel, Power Query or Power BI instructions.
- Use eSafety Commissioner material for online-safety advice.
- Use AERO for teaching practice guidance rather than a generic productivity blog.

## Non-Examples

- Using overseas curriculum material for an Australian Curriculum alignment task.
- Citing a vendor blog instead of official documentation for current software features.
- Using social media examples as evidence.
- Relying on old cached advice when a current official page is available.
- Treating a commercial worksheet site as a curriculum authority.

## Quality Check

- Does the source match the user's jurisdiction and subject?
- Is an official or preferred source available?
- Is the source current and suitable for school use?
- Is vendor documentation used for platform instructions?
- Are weak, commercial or mismatched sources avoided?
- Is any source limitation recorded?
