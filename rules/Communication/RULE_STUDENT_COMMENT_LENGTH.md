# Rule: Student Comment Length

## Purpose

Keep student feedback, report-style comments and LMS notes concise enough to be read and acted on, while still giving useful evidence and a next step.

## Applies To

- student feedback comments
- rubric comments
- report comment source material
- LMS feedback boxes
- quick formative feedback
- short parent-facing learning updates

## Rule Statement

The agent must match comment length to the platform, purpose and student's ability to act on the feedback.

## Requirements

- Follow any character or word limit supplied by the user.
- Keep one comment focused on one main strength and one main next step unless more is requested.
- Use short sentences.
- Avoid repeating the rubric unless the comment adds evidence or action.
- Do not include every error in a short comment.
- Use student-friendly wording for student-facing comments.
- Keep parent-facing comments clear and jargon-free.
- Use longer comments only when the task, reporting format or user request justifies it.
- Preserve privacy and avoid unnecessary personal detail.

## Decision Logic

If no length is supplied, default to a concise comment of one to three sentences. If the user requests report comments, follow the school or platform limit if supplied. If feedback needs more detail, use a short structured format rather than a dense paragraph.

## Examples

- "You have used a clear data table and identified the trend. Next, use two data points to support your conclusion."
- "Your code uses a loop correctly. Next, add comments to explain the purpose of each section."
- "The explanation is accurate but needs a labelled diagram to make the process clearer."

## Non-Examples

- A paragraph listing ten corrections after a short formative task.
- "Good job" with no evidence or next step.
- A report comment that exceeds the platform limit.
- Repeating the whole rubric descriptor as feedback.
- Including sensitive behaviour details in a general comment.

## Quality Check

- Does the comment meet the requested limit?
- Is there one clear strength or evidence point?
- Is there one clear next step?
- Is the language suitable for the audience?
- Is unnecessary detail removed?
- Is privacy protected?
