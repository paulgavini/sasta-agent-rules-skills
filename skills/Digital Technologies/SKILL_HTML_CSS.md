# Skill: HTML CSS

## Purpose

This skill allows the agent to create beginner web-development resources that separate structure, presentation and content clearly.

## When To Use

Use this skill when the user asks for:

- HTML or CSS tutorials
- web page starter code
- website assessment scaffolds
- explanations of tags, elements, selectors or styling
- accessible web layout guidance
- debugging help for HTML or CSS

## Inputs

- year level and prior web-development experience
- target concept: HTML structure, CSS styling, layout, accessibility or responsive design
- required page content
- editor or platform: browser, CodePen, Replit, VS Code, LMS or static file
- whether JavaScript is allowed
- device and browser constraints
- assessment requirements

## Process

1. Define the page purpose and audience.
2. Plan the content structure before visual styling.
3. Use semantic HTML where appropriate.
4. Add CSS in a clear, limited way tied to the learning goal.
5. Keep file structure simple for beginners.
6. Separate HTML, CSS and expected result.
7. Include accessibility checks for headings, links, contrast, images and readable layout.
8. Debug by checking file links, selectors, spelling, nesting and browser refresh.
9. Avoid unnecessary frameworks unless requested.

## Output

```markdown
# HTML/CSS Activity

## Goal
## File Setup
## HTML
## CSS
## Test In The Browser
## Accessibility Checks
## Extension
```

## Quality Criteria

- HTML and CSS are separated or clearly labelled.
- Code is copyable and syntactically plausible.
- The example is small enough to inspect.
- Accessibility is considered.
- The task does not require unapproved accounts or installs.
- Extensions are optional.
