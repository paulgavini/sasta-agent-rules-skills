# Rule: Feedback Format

## Purpose

Ensure student feedback is specific, evidence-based, respectful and useful for next learning steps.

## Applies To

- written feedback comments
- rubric feedback and marking notes
- report comment source material
- formative checkpoints and draft feedback
- parent-facing explanations of assessment progress

## Rule Statement

The agent must format feedback so it identifies evidence, explains the learning implication and gives a manageable next step.

## Requirements

- Start from the assessment criteria, learning intention or success criteria.
- Refer to specific evidence from the student's work.
- Name one or two strengths before prioritising improvement when appropriate.
- Give actionable next steps students can understand and attempt.
- Keep tone respectful, professional and student-centred.
- Avoid comments about personality, effort or attitude unless directly relevant and appropriate.
- Do not overload students with too many corrections at once.
- Use Australian English and year-level appropriate language.
- Keep privacy in mind when examples include identifiable student information.

## Decision Logic

For formative feedback, prioritise the next step that will most improve learning. For summative feedback, explain the judgement using evidence and standards, then give a forward-looking improvement point. For parent-facing feedback, use plain language and avoid unnecessary technical marking language.

## Example Format

```markdown
Strength:
Evidence:
Next step:
```

For compact comments:

```markdown
You have shown [evidence/strength]. To improve, [specific next action].
```

## Examples

- "You identified the independent variable and collected repeated measurements. Next, explain how your controlled variables made the test fair."
- "Your code uses selection correctly for valid inputs. Next, add a check for invalid input so the program handles errors."
- "The paragraph uses relevant evidence, but the explanation needs to connect the evidence back to the claim."

## Non-Examples

- "Great work, try harder next time."
- "This is poor."
- "You are careless with graphs."
- "Improve your analysis" without saying how.
- Listing every minor error in a way that hides the main next step.

## Quality Check

- Is feedback linked to evidence and criteria?
- Is the next step specific and manageable?
- Is the tone respectful and professional?
- Is the comment concise enough for the context?
- Does it avoid unsupported personal judgement?
- Is privacy protected?
