# Rule: Frog, Stile And Daymap Context

## Purpose

Ensure school-platform instructions for Frog, Stile and Daymap are practical, privacy-aware and clearly separated by platform purpose.

## Applies To

- LMS-ready instructions
- lesson and assessment posting guidance
- parent, student and staff communication
- Stile lesson customisation notes
- Daymap task, due date, grade or attendance references
- Frog resource, homework or communication workflows

## Rule Statement

The agent must not treat Frog, Stile and Daymap as interchangeable; it must use each platform according to the supplied school context and avoid guessing local settings, permissions or workflows.

## Requirements

- Use the platform named by the user and do not substitute another platform without permission.
- Distinguish broad LMS/resource posting from Stile science lessons and Daymap learning/student-management records.
- Do not invent local menu names, URLs, permissions, notification settings or admin workflows.
- Keep student-facing platform instructions short, ordered and action-based.
- Include due dates, submission expectations and file formats only when known or supplied.
- Avoid exposing student grades, attendance, behaviour or personal information in inappropriate channels.
- Use Daymap-related references carefully because schools configure access and parent/student portals differently.
- Treat Stile AI or feedback features as teacher-support tools requiring teacher judgement, not automatic final marking.
- State when school policy, local admin settings or ICT support must be checked.
- Provide platform-neutral wording when local configuration is unknown.

## Decision Logic

If the user supplies local platform details, follow them exactly. If details are missing, write generic instructions that a teacher can adapt rather than inventing clicks. If the output includes grades, attendance, behaviour, assessment feedback or personal data, prioritise privacy and role-appropriate access.

## Examples

- "Post the worksheet in your class LMS using the school's usual assignment workflow" when the exact Frog configuration is unknown.
- "In Stile, adapt the activity prompts and review student responses before sending feedback."
- "Record due dates and assessment information in Daymap according to the school's usual process."

## Non-Examples

- Claiming a specific Daymap menu path without a supplied school guide.
- Using Stile for attendance records.
- Posting individual grades in a public class announcement.
- Assuming parents, students and staff see the same platform view.
- Treating AI-generated feedback in Stile as final teacher feedback without review.

## Quality Check

- Is the correct platform named and used for the right purpose?
- Are local settings avoided unless supplied?
- Is privacy protected for student records and feedback?
- Are instructions short enough for LMS posting?
- Is teacher judgement preserved for feedback and assessment?
- Are required school-policy or ICT checks identified?
