# Rule: Table Style

## Purpose

Ensure tables are used only when they improve readability, comparison or exactness, and remain accessible when copied, exported or converted.

## Applies To

- rubrics and marking guides
- lesson sequence tables
- equipment and preparation lists
- data tables and practical templates
- source registers and progress logs
- Word-ready and Markdown documents

## Rule Statement

The agent must create simple, readable and accessible tables with clear headings and no unnecessary layout complexity.

## Requirements

- Use a table only when rows and columns make the information easier to compare or scan.
- Prefer a list or short paragraph when there are only one or two items.
- Give every table clear column headings.
- Keep cell content concise.
- Avoid nested tables, merged cells, split cells and blank structural cells.
- Avoid using tables only for page layout.
- Keep table width manageable for the expected destination.
- Place the table near the text that introduces it.
- Make tables self-explanatory but still refer to them in surrounding text where needed.
- For student worksheets, avoid tables that are too dense for the year level.

## Decision Logic

Use tables for exact values, criteria, schedules, comparisons, evidence maps and structured registers. Use lists for instructions, checklists and short grouped information.

## Examples

- A rubric table with `Criterion`, `Expected evidence`, `Feedback notes`.
- A lesson sequence table with `Lesson`, `Focus`, `Activity`, `Evidence`.
- A practical equipment table with `Item`, `Quantity`, `Notes`.

## Non-Examples

- A one-row table for a single note.
- A table used only to place text beside an image.
- A rubric with merged cells that will not copy cleanly.
- A wide table with long paragraphs in every cell.
- A nested table inside a source register.

## Quality Check

- Does the table make the information easier to use?
- Are headings clear?
- Is the table simple enough to export?
- Are cells concise?
- Are nested or merged structures avoided?
- Would a list work better?
