# Skill: Robotics Programming

## Purpose

This skill allows the agent to create beginner robotics programming activities that connect sensors, decisions, actuators and testing.

## When To Use

Use this skill when the user asks for:

- robotics lessons
- line-following, obstacle-avoidance or remote-control tasks
- motor, servo or actuator programming
- robot troubleshooting
- micro:bit, Arduino-compatible or kit-based robotics tasks
- robotics assessment scaffolds

## Inputs

- robot platform or kit
- controller and programming environment
- available sensors and actuators
- movement goal or challenge
- power source and classroom space
- student programming level
- safety and supervision constraints

## Process

1. Define the robot behaviour in observable terms.
2. Identify inputs, processing decisions and outputs.
3. Test sensors before adding movement.
4. Test motors or servos at low speed before combining behaviours.
5. Build a simple control loop.
6. Add one condition or behaviour at a time.
7. Use safe testing zones and stop conditions.
8. Debug by separating sensor readings, motor direction, timing, battery level and code logic.
9. Include reflection on how feedback from sensors changes robot behaviour.

## Output

```markdown
# Robotics Activity

## Challenge
## Hardware
## Inputs And Outputs
## Algorithm
## Code Or Blocks
## Test Plan
## Safety Notes
```

## Quality Criteria

- The robot behaviour is testable.
- Sensors and actuators are identified separately.
- Movement is tested safely.
- The control logic is visible.
- Troubleshooting separates hardware from code.
- The task fits the available kit and classroom space.
