# Skill: Differentiation

## Purpose

This skill allows the agent to add practical support, access and extension options to lesson materials while preserving the core learning goal and assessment integrity.

## When To Use

Use this skill when the user asks for:

- differentiated lesson options
- support and extension tasks
- access adjustments for a lesson or worksheet
- mixed-ability class planning
- teacher-facing differentiation notes
- student-facing scaffolded versions of a task
- inclusive classroom adaptations

## Inputs

- year level, subject and topic
- core learning goal or task
- student needs supplied by the user
- class profile, if available
- required evidence or assessment conditions
- available time, resources, equipment and platforms
- accessibility, language, literacy or numeracy considerations
- local school support or adjustment requirements if supplied

## Process

1. Identify the non-negotiable core learning goal.
2. Separate access supports from extension options.
3. If the user supplies specific student needs, tailor suggestions to those needs without adding labels or assumptions.
4. If no profile is supplied, provide general options for vocabulary, structure, modelling, pacing, grouping, examples and extension.
5. Keep support connected to the same learning goal where possible.
6. Design extension to deepen, transfer, justify, evaluate or create, rather than simply adding more of the same.
7. Distinguish student-facing supports from teacher-facing notes.
8. For assessment tasks, protect the evidence students must produce independently.
9. Flag formal adjustments, disability supports or modified assessment conditions as teacher or school decisions.
10. Check that options are manageable for one teacher in the stated context.

## Output

```markdown
# Differentiation Options

## Core Learning Goal

## Support

## Extension

## Accessibility And Language

## Teacher Notes

## Assessment Integrity Notes
```

Adjust headings to match the task and omit sections that are not relevant.

## Quality Criteria

- The core learning goal remains clear.
- Support helps students access the intended learning.
- Extension deepens learning rather than adding volume.
- Wording is respectful and inclusive.
- Teacher-facing and student-facing notes are separated.
- Formal adjustment needs are not invented or overclaimed.
- Options are practical for the classroom context.

## Example User Requests

- `Differentiate this Year 8 worksheet.`
- `Add support and extension options to this practical.`
- `Make this task accessible for mixed literacy levels.`
- `Create teacher notes for differentiation.`

## Failure Modes

- Lowering expectations without preserving access to the main learning.
- Making extension just extra questions.
- Using deficit labels in student-facing content.
- Inventing individual adjustments the user did not provide.
- Providing support that compromises assessment evidence.
- Suggesting options that are unrealistic for the stated lesson time.

## Completion Checklist

- Is the core goal explicit?
- Are support and extension clearly separated?
- Is language respectful?
- Are assessment limits protected?
- Are teacher decisions and formal adjustments flagged?
- Are options classroom-ready?
