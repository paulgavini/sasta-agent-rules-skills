# Rule: Single Point Rubric

## Purpose

Ensure single-point rubrics focus students and teachers on expected evidence, strengths and next steps without unnecessary performance-level clutter.

## Applies To

- formative assessment
- drafting, conferencing and feedback cycles
- short practical reports, coding tasks and design tasks
- success criteria and self-assessment
- tasks where feedback is more useful than a full A-E rubric

## Rule Statement

The agent must use single-point rubrics when the task benefits from a clear expected standard plus space for evidence-based strengths and improvements.

## Requirements

- Define the expected evidence or success criterion in the centre column.
- Include space for strengths and next steps or concerns.
- Keep criteria few enough for students to use.
- Use student-friendly wording when students will see the rubric.
- Align criteria to curriculum, standards or task evidence.
- Avoid vague criteria such as "good effort" unless effort is explicitly assessed.
- Use observable evidence rather than hidden thinking.
- Keep feedback focused on learning improvement.
- Convert to an A-E rubric only when graded performance levels are required.

## Decision Logic

Use a single-point rubric for formative feedback, drafts, checkpoints, peer review and self-assessment. Use an A-E rubric when the teacher needs grade-level descriptors. If the user asks for both grading and feedback, pair the single-point structure with a separate grade judgement or marking guide.

## Example Format

```markdown
| Strengths shown | Expected evidence | Next step |
| --- | --- | --- |
|  |  |  |
```

## Examples

- A draft practical report rubric lists expected evidence for question, method, data, conclusion and evaluation.
- A coding task rubric gives space to note working features and debugging priorities.
- A peer review rubric uses student-friendly criteria tied to the task.
- A teacher feedback sheet records evidence and next step without assigning a grade.

## Non-Examples

- A single-point rubric with ten criteria for a short task.
- Expected evidence written in teacher-only jargon for Year 8 students.
- Strengths and next steps that are not linked to criteria.
- Using a single-point rubric when the user explicitly needs A-E grade descriptors.
- Including personality comments instead of evidence.

## Quality Check

- Is the expected evidence clear?
- Are criteria few and useful?
- Are strengths and next steps evidence-based?
- Is the language suitable for the audience?
- Does the format match formative or feedback use?
- Is grading kept separate unless requested?
