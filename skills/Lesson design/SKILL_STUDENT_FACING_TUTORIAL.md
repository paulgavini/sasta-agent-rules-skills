# Skill: Student Facing Tutorial

## Purpose

This skill allows the agent to create student-ready tutorials, worksheets and guided learning materials that are clear, scaffolded, age-appropriate and suitable for classroom or LMS use.

## When To Use

Use this skill when the user asks for:

- a student tutorial
- a worksheet or guided task
- a step-by-step learning activity
- a practical investigation scaffold
- a coding or software tutorial
- a revision or catch-up resource
- instructions that students will read directly

## Inputs

- year level or SACE stage
- subject, topic and learning goal
- expected prior knowledge
- platform, software, equipment or materials
- lesson length or expected completion time
- required output or submission
- support, extension or accessibility needs
- any supplied curriculum, assessment or school requirements

## Process

1. Identify what students need to learn, do and produce.
2. Choose a structure that matches the task: explanation, worked example, guided practice, independent task, reflection or submission.
3. Write the opening in student-facing language with the learning goal and immediate task clear.
4. Explain essential vocabulary before students need to use it.
5. Break the task into numbered steps or short sections.
6. Include modelling or a worked example when the process is new or complex.
7. Add checks for understanding, practice questions or reflection prompts where useful.
8. Separate required work from extension.
9. Keep teacher notes out of the student instructions unless clearly labelled for teacher use.
10. Check accessibility, tone, length, formatting and copy-paste readiness for the intended platform.

## Output

```markdown
# Tutorial Title

## Goal

## What You Need

## Key Terms

## Step 1

## Step 2

## Practice

## Check Your Work

## Extension

## What To Submit
```

Adjust headings to match the user's requested format and omit sections that do not help.

## Quality Criteria

- Students can tell what to do first.
- Instructions are direct, plain and age-appropriate.
- The task is scaffolded without doing the thinking for students.
- Required and extension work are clearly separated.
- Technical vocabulary is explained when needed.
- Submission or completion expectations are explicit.
- Formatting is suitable for copy-paste into LMS, Word or classroom handouts.

## Example User Requests

- `Write a Year 8 student tutorial on variables in fair testing.`
- `Create a beginner Python loop worksheet.`
- `Turn these notes into a student-facing practical guide.`
- `Make this suitable for posting in Stile.`

## Failure Modes

- Writing teacher planning notes inside student instructions.
- Starting with a long explanation before the task is visible.
- Using curriculum jargon students do not need.
- Giving extension tasks without labelling them as optional or extension.
- Assuming students know the platform, equipment or vocabulary.
- Making the tutorial so scaffolded that students only copy steps.

## Completion Checklist

- Is the student task clear?
- Are steps sequenced logically?
- Is the tone direct and respectful?
- Are vocabulary and examples sufficient?
- Are common mistakes or checks included where useful?
- Is the output ready for the requested platform?
