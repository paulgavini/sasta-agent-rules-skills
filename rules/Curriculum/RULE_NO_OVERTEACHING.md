# Rule: No Overteaching

## Purpose

Keep resources focused on the requested learning goal, year level and assessment evidence rather than expanding into unnecessary theory, extra curriculum content or advanced edge cases.

## Applies To

- explanations, worked examples and tutorials
- lesson plans, worksheets and slide content
- assessment tasks, rubrics and marking guides
- curriculum and SACE-aligned resources
- feedback, report comments and parent-facing explanations

## Rule Statement

The agent must not add extra content, complexity or advanced concepts unless they directly support the requested outcome, safety, accessibility, accuracy or a clearly labelled extension.

## Requirements

- Start from the user's stated task, year level, topic and deliverable.
- Include only the concepts students need to complete the learning or assessment goal.
- Keep background theory brief and purposeful.
- Separate optional extension from required learning.
- Avoid adding multiple methods, edge cases or exceptions when one clear method is enough for the audience.
- Do not turn a worksheet, feedback comment or LMS post into a mini-textbook.
- Use scaffolding to reduce unnecessary load, not to add more tasks than students can reasonably complete.
- Preserve curriculum accuracy while avoiding content beyond the intended year level.
- Ask or state an assumption when the desired depth is unclear and overteaching would affect usability.

## Decision Logic

If extra material is interesting but not needed, leave it out. If extra material is needed for safety, misconception prevention, assessment validity or accessibility, include it briefly. If the user asks for extension, put extension after the core task and label it clearly.

## Examples

- A Year 8 lesson on variables should define independent, dependent and controlled variables without adding formal statistical inference.
- A beginner Python task should teach the required construct before introducing alternative advanced syntax.
- A SACE assessment may need precise standards language, but student instructions should still stay focused on what students must do.
- Extension questions should not become hidden core requirements.

## Non-Examples

- Adding senior secondary theory to a junior worksheet without being asked.
- Including every possible formula variation in a worked example for beginners.
- Expanding a parent email into a detailed pedagogical essay.
- Adding assessment criteria that are not aligned to the task evidence.
- Treating "more information" as automatically better teaching.

## Quality Check

- Does every section support the requested outcome?
- Is the core learning clear before any extension appears?
- Has unnecessary theory, duplication or edge-case detail been removed?
- Is the pitch suitable for the year level or SACE stage?
- Would the teacher need to cut substantial content before using it?
- Are optional additions clearly labelled?
