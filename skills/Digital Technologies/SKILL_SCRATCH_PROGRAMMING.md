# Skill: Scratch Programming

## Purpose

This skill allows the agent to create Scratch activities, tutorials, debugging guides and assessment scaffolds that use block-based programming accurately and safely.

## When To Use

Use this skill when the user asks for:

- a Scratch lesson or worksheet
- a Scratch game, animation or simulation task
- sprite, event, variable, broadcast or clone guidance
- Scratch debugging help
- a beginner programming activity using blocks
- a Scratch assessment scaffold or rubric support

## Inputs

- year level and prior Scratch experience
- project type: animation, game, quiz, story, simulation or skill drill
- required sprites, backdrops, variables or controls
- online or offline Scratch access
- whether students have accounts or teacher-managed classes
- assessment conditions and required independence
- required output format

## Process

1. Define the visible behaviour students should create.
2. Identify sprites, backdrops, events and variables.
3. Plan the project with a short algorithm, storyboard or event table.
4. Build the smallest working version first.
5. Add one feature at a time.
6. Include predict-run-modify checks.
7. Debug by sprite, event and block order.
8. Add privacy and sharing cautions when accounts, studios or comments are involved.
9. Separate core requirements from extension features.

## Output

```markdown
# Scratch Activity

## Goal
## Setup
## Blocks Or Steps
## Test
## Debugging Checks
## Extension
```

Adjust headings to suit the user's requested format.

## Quality Criteria

- Scratch terms are accurate.
- Sprite ownership and event triggers are clear.
- Students can test visible behaviour.
- Account and sharing assumptions are not invented.
- Extension does not obscure the core task.
